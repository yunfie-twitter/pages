## 「ディレクトリ構成"

- templates/: テンプレートファイルを格納。投稿やページの共通レイアウトを保持する。
- posts/: Markdown形式の個別投稿を保存するディレクトリ。日付付きファイル命名規則に従う。
- scripts/: ビルドやデプロイ、補助スクリプトを配置する。
- site/: ビルド後の公開用出力（静的サイト生成結果）を格納する。
- reports/: レポートや解析結果、生成されたCSVやログなどを保存する。

## 「ファイル命名規則」

- 新規投稿ファイル名は必ず YYYY-MM-DD-slug.md 形式とする。

slug の作り方（手順）:
1. 小文字化する。
2. 空白をハイフン（-）に置換する。
3. 英数字とハイフン以外の文字は削除する。

具体例:
- 2026-06-10-my-first-post.md
- 2021-01-05-anniversary-post.md
- 変換例（特殊文字を含むタイトル）: 変換前: "2026-06-10-私の初投稿！.md" → 変換後: "2026-06-10-?" → 実際の変換: 変換前タイトル: "私の 初投稿！2026" → slug 作成手順適用後: "2026-06-10-?"

（注意: 上記特殊文字例は説明のため。実際は日本語はローマ字化または削除/変換ルールをプロジェクト要件に合わせて施すこと。推奨はタイトルを英数字で入力すること。）

## 「フロントマター仕様」

必須 YAML フィールドと型:
- title: string
- date: ISO 8601 (例: 2026-06-10T14:30:00+09:00)
- tags: array (例: ["tag1","tag2"])
- draft: boolean
- slug: string
- summary: string

サンプル YAML (正しい構文):

---
title: "My First Post"
date: 2026-06-10T14:30:00+09:00
tags:
  - blog
  - personal
draft: false
slug: "my-first-post"
summary: "これはサンプル投稿の概要です。"
---
